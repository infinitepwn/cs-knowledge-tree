# OSPF协议

> 来源：`计算机网络 / 网络层/OSPF协议.md`

## 定义

OSPF（Open Shortest Path First，开放最短路径优先）是基于链路状态的内部网关协议。

OSPF 使用 Dijkstra 算法计算最短路径。

采用泛洪法向路由器端口的所有路由器发送信息，然后继续推广
OSPF 基于链路状态协议,每个路由器获得全网拓扑

## 区域划分

OSPF还会把AS再划分成区域

利用泛洪法交换链路状态信息局限于每一个区域而不是整个的自治系统，减少了整个网络上的通信量

![image.png](https://raw.githubusercontent.com/infinitepwn/note_picbed/main/20260615215631452.png)

### 层次结构的区域划分
在上层的区域叫做主干区域，标识符为0.0.0.0，用来连通其他在下面的区域

从其他区域来的消息都由区域边界路由器进行概括

在主干区域内的路由器叫做主干路由器，主干路由器

主干路路由器也可以是区域边界路由器

### 分组

Hello 分组用于发现和维持邻居关系。

## 与 RIP 的区别

OSPF是网络层协议，直接用IP数据报传输
而RIP是应用层协议，在运输层用UDP
